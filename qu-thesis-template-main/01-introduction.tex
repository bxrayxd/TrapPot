\begin{center}
{\LARGE\textbf{INTRODUCTION}}
\end{center}


Saudi Arabia is adopting Internet of Things (IoT) devices in smart-city projects and other digital transformation initiatives under Vision 2030. These devices expand connectivity, but many of them remain exposed because of weak credentials, limited patching, and insecure remote access services. The 2016 Mirai botnet demonstrated the scale of this risk by compromising more than 600,000 IoT devices worldwide and using them to cause major internet outages \cite{203628}. This incident shows that insecure IoT devices are not only a technical issue, but also a threat to companies, infrastructure, and public digital services.

Honeypots provide a controlled way to attract attackers and study their behavior without exposing production systems. At the same time, artificial intelligence can help classify attack patterns and reduce the amount of manual analysis required from security teams. TrapPot combines these two ideas in one system. It uses a Cowrie-based SSH and Telnet honeypot to capture attacker interaction, Zeek to summarize network traffic, a Random Forest model to classify traffic behavior, and the ELK Stack to store and visualize the collected evidence. The goal is to support practical IoT threat monitoring and make attack behavior easier to understand in an academic and local cybersecurity context.

\section{Challenges}
\label{sec:challenges}
% ============================================================


Several challenges appear when developing an IoT honeypot system. First, the project requires knowledge of cybersecurity, networking, containerization, log management, and machine learning, which makes it technically demanding for an undergraduate team. Second, IoT honeypot projects have fewer practical tutorials and case studies than more common IT topics, so the team had to rely on academic papers, official documentation, and direct testing. Third, a honeypot must be isolated carefully. If it is connected to the wrong network or configured without containment, attackers could misuse it as an entry point or as a platform for further activity. Finally, the system must handle continuous log collection, storage, feature extraction, and classification without losing the relationship between attacker behavior and network traffic.

TrapPot addresses these challenges by using established open-source tools, a Docker-based deployment, and strict lab-network assumptions. Cowrie captures attacker interaction, Zeek produces structured network logs, the Random Forest detector classifies traffic, and ELK provides centralized monitoring. This design reduces the amount of custom infrastructure needed while still allowing the project to study real attack behavior in a controlled way.

\section{Study Scope}
\label{sec:study_scope}
% ============================================================

This research project aims to create an AI-based honeypot system that emulates Telnet and SSH services to monitor IoT-targeted attacks inside a standalone network segment. The scope is limited to Random Forest classification for five classes: brute-force, scanning, malware download, command execution, and normal traffic. The study also keeps the original 30-day collection objective and the minimum 99\% classification accuracy target. The implemented system operates as a single-node proof of concept with network isolation to avoid compromising production systems. Advanced protocols such as HTTP, MQTT, and CoAP, deep learning methods, distributed deployments, and production-ready hardening are outside the scope of this phase and are reserved for future work.

Random Forest has already shown high accuracy and low latency for classifying honeypot-captured IoT botnet attacks, making it a suitable choice for TrapPot's classifier \cite{lee2021classification}. Telnet and SSH services are also primary targets in Mirai-like IoT botnet activity, which motivates the focus on these protocols and the selected attack types \cite{famera2025mirai,zhang2023identification}.

\section{Goals and Objectives}
\label{sec:goals_and_objectives}
% ============================================================

The project aims to improve IoT security readiness by observing attacker behavior, analyzing network traffic, and classifying suspicious activity. The main objectives are:

\begin{itemize}
    \item \textbf{Understanding IoT Attack Behavior:} Investigate attacker interactions to understand how IoT-like systems are discovered, probed, and exploited. This includes analyzing commands, login attempts, and traffic patterns produced during attack sessions.
    \item \textbf{Improving IoT Attack Classification:} Develop an AI-based classifier that can categorize brute-force attempts, scanning behavior, normal traffic, malware-related activity, and command execution. A core objective is to achieve at least 99 percent classification accuracy on the prepared dataset.
    \item \textbf{Supporting Proactive Security Measures:} Provide earlier visibility into malicious behavior so that security teams can apply preventive measures before damage occurs.
    \item \textbf{Enhancing IoT Threat Awareness:} Present captured activity in a structured way so researchers, administrators, and organizations can understand common attack methods and trends.
    \item \textbf{Reducing the Attack Surface of IoT Environments:} Use findings from observed attack behavior to guide practical recommendations, such as improving access controls and reducing unnecessary service exposure.
\end{itemize}

Together, these objectives support better decision-making and stronger security practices for IoT environments facing changing cyberattack patterns.
