To provide clarity for the system design discussed in the Methodology chapter, this section defines the fundamental technologies and concepts that form the building blocks of the TrapPot framework. \subsubsection{HoneyPots}
\label{subsub:HoneyPots}
A honeypot is a security mechanism designed to act as a decoy system, intentionally exposed to attract and engage attackers. By mimicking vulnerable services, it allows for the capture and study of malicious activity without risking production assets. Its primary value lies in gathering intelligence on attack methodologies, tools, and motivations \cite{9268794}. The integration of honeypots with machine learning has become a significant trend, as they provide the rich, real-world data needed to effectively train detection models. This is especially common in the IoT domain, where honeypots are widely used to collect malware samples for analysis \cite{9268794}.

\subsubsection{IoT HoneyPots}
\label{subsub:IoT HoneyPots}
IoT honeypots are a specialized category of decoy systems designed to emulate the specific hardware and software of Internet of Things devices, such as smart cameras, sensors, and routers. They are a critical security tool for understanding the unique threat landscape targeting IoT, Industrial IoT (IIoT), and Cyber-Physical Systems (CPS) \cite{9520645}. Due to the inherent vulnerabilities and limitations of real IoT devices, honeypots serve as an effective mechanism to attract attackers, gather intelligence on IoT-specific malware and attack patterns, and complement other security defenses like firewalls and intrusion detection systems. Their deployment is essential for researching and defending the rapidly expanding IoT ecosystem \cite{9520645}.

\subsubsection{Machine Learning for Cybersecurity}
\label{subsub:Machine Learning for Cybersecurity}
Machine Learning (ML) and its sub-field, Deep Learning (DL), have become fundamental approaches for developing modern intrusion detection and cyber defense systems. These techniques help systems to move beyond fixed methods, signature-based detection by learning from data to identify complex patterns and anomalies indicative of malicious activity. By analyzing network traffic and system logs, ML/DL algorithms can automatically classify behaviors, making them highly effective for detecting evolving cyber-attacks \cite{9268794}. The application of these data-driven approaches is crucial for building proactive and adaptive security solutions like TrapPot, which relies on ML to intelligently classify attacks captured by its honeypot sensors.

\subsubsection{Key Tools & Protocols}
\label{subsub:Key Tools & Protocols}
The implementation of the TrapPot system relies on a foundation of specific software tools and network protocols. This subsection provides a concise overview of these core technologies, which are instrumental in the data capture, analysis, and visualization processes detailed in the following Methodology chapter.

Cowrie Honeypot: Cowrie is an open-source, medium-interaction honeypot that emulates Secure Shell (SSH) and Telnet services. Its primary function is to act as a deceptive trap, luring attackers in to log brute-force attempts and capture shell interaction sequences, including commands executed and files uploaded. This makes it a valuable tool for gathering intelligence on attacker methodologies targeting these common network protocols \cite{9071014}.

ELK Stack: The ELK Stack is a powerful, open-source platform for log management and data analytics, composed of three core components: Elasticsearch, a distributed search and analytics engine; Logstash, a server-side data processing pipeline; and Kibana, a visualization and dashboarding tool. Together, they provide a comprehensive solution for centralized logging, real-time data processing, and intuitive data visualization, which is essential for analyzing the high volume of security events generated by honeypots \cite{9001068}.

Telnet & SSH Protocols: Secure Shell (SSH) and Telnet are network protocols designed for remote command-line access and management of devices. Their widespread use has made them primary targets for cyber-attackers, especially in the IoT landscape, where they are often exploited using brute-force attacks to gain unauthorized control. This makes the emulation of these services a critical function for honeypots like Cowrie to capture relevant attack data \cite{9558948}.
